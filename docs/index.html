"What date is 5 business days after Dec 22nd in the UK?" "How many working days are in this month?" "Is Friday a working day in Saudi Arabia?" LLMs answer these confidently and often wrongly — they miscount, forget holidays, and assume every country's weekend is Saturday/Sunday. bizdays gives an agent the deterministic answer, showing the weekend rule and the exact holidays it skipped. MCP server + typed TypeScript library.

Method: 30 business-day questions across 12 countries on the 2025 calendar, asked of a frontier model with no tools, scored against ground truth computed from authoritative sources (date-holidays for public holidays, Unicode CLDR for weekends). The model does fine on small "add N days" hops but breaks on counting a full month and on non-Western weekends (it flipped a UAE Friday and Saudi National Day). bizdays is deterministic — correct by construction. Reproduce or extend it on any model with the harness in the repo (bench/).

Why it's right when libraries are wrong

Two failure modes, both fixed:

Weekends ≠ Sat/Sun

Resolved per country from Unicode CLDR — Fri/Sat in Saudi Arabia, Egypt, Israel; Fri in Iran; Sun in India; Sat/Sun in the UAE (post-2022) — with a curated override layer for cases CLDR gets wrong (Bangladesh, Nepal).

Public holidays only

Off-the-shelf holiday data counts observances like Valentine's Day, Tax Day and Christmas Eve as holidays. Those aren't days off. bizdays counts only real public holidays, so deadlines don't drift.

Use it

As an MCP server, in your client config:

{
  "mcpServers": {
    "bizdays": { "command": "npx", "args": ["-y", "@qinisolabs/bizdays"] }
  }
}

As a TypeScript library:

npm install @qinisolabs/bizdays

import { addBusinessDays, countBusinessDays, isBusinessDay } from "@qinisolabs/bizdays";

addBusinessDays("2025-12-22", 5, "GB").result;   // "2025-12-31" (skips Christmas + Boxing Day)
isBusinessDay("2025-06-13", "SA").isBusinessDay;  // false — Friday is weekend in Saudi Arabia
isBusinessDay("2025-02-14", "US").isBusinessDay;  // true  — Valentine's Day is not a public holiday

Tools

add_business_days

Add or subtract N working days to a date, skipping weekends + public holidays.

count_business_days

Count working days between two dates, with a weekend/holiday breakdown.

is_business_day

Is a date a working day? If not, the precise reason and holiday name.

next_business_day

First working day strictly after a date.

previous_business_day

Last working day strictly before a date.

country_rule

Weekend rule + source, whether holidays apply, and a confidence flag.

Coverage

Public holidays for 206 countries and subdivisions (via date-holidays, filtered to public holidays only). Weekends from CLDR with curated corrections. A Tier-1 set of ~65 countries is verified; others work but are flagged unverified so the agent knows. Where no holiday calendar exists (e.g. Qatar, Kuwait, Oman), the correct weekend is still applied and the response says so.

Scope

bizdays computes working days from weekends and public holidays. It does not include regional or optional holidays unless you pass them via extraHolidays, and it makes no network calls. Dates are plain YYYY-MM-DD, handled in UTC.
